##BASE DE DATOS:
Conjunto de datos definidos bajo una misma realidad o ambito almacenados de forma sistematica y estructurada en una o mas tablas para su posterior recuperacion.
La bd tambien tiene vistas indices, funciones, procedimientos, registro de modificaciones, entre otros.



##SQL O NO SQL
##############################################################
SQL => STRUCTURED QUERY LANGUAGE -------- MODELO RELACIONAL
    EN SQL LOS DATOS SE GUARDAN COMO SI FUERAN UNA PLANILLA DE CALCULO, ES DECIR COLUMNAS Y FILAS.
        Y PARA PODER RELACIONAR LOS DATOS QUE SE ENCUENTRAN ALMACENADOS EN 2 TABLAS SE HACE USO DE LLAVE PRIMARIA Y FORANEA.

    LAS RELACIONES TIENE 2 CARACTERISTICAS IMPORTANTES:
        GRADO: REPRESENTA NUMERO DE ENTIDADES(TABLAS) QUE PARTICIPAN EN LA RELACION
        CARDINALIDAD: NUMERO DE ELEMENTOS (REGISTROS DE TABLA) DE UN TIPO (UNA TABLA O ENTIDAD) QUE SE CONECTAN CON UN ELEMENTO DE OTRO TIPO (OTRA TABLA O ENTIDAD)

    RELACIONES:
        MUCHOS A MUCHOS: (N:M)
            -UN ALUMNO CURSA UNA MATERIA, Y UNA MATERIA PUEDE SER CURSADA POR MUCHOS ALUMNOS
            -VENTA INCLUYE MUCHOS PRODUCTOS, PRODUCTO INCLUYE MUCHAS VENTAS

        UNO A MUCHOS: (1:M)
            -1 HABITACION PERTENECE A TIPO DE HABITACION
            -CLIENTE POSEE CUENTA BANCARIA

        RELACION UNO A UNO: (1:1)


    TIPOS DE DATOS:
        -CADENA:
            -VARCHAR LONGITUD VARIABLE
            -CHAR LONGITUD FIJA
        -NUMERICO ENTERO
            -TINYINY
            -SMALLINT
            -INT
            -BIGINT
        -NUMERICO DECIMAL
            -FLOAT
            -DOUBLE
        -FECHA
            -DATETIME
        -BLOB
            -DATO BINARIO

TRANSACIONES ACID => 
    ES UNA SECUENCIA DE OPERACIONES(TAREAS) REALIZADAS COMO UNA SOLA UNIDAD DE TRABAJO, UN SOLO PROCESO QUE SIGUE SOLO 2 POSIBLES CAMINOS:
        -SE REALIZAN TODAS
        -O NO SE REALIZA NINGUNA(CASO DE FALLA)
    UNA TRANSACCION DEBE SER CAPAZ DE MANTENER LA INTEGRIDAD DE LOS DATOS.

    -ATOMICAS: TU REALIZAS CONSULTAS QUE SE COMPONE DE VARIAS, PERO CUANDO SE LAS ENVIAS AL SERVER SE VA ENCARGAR DE EJECUTARLAS COMO UNA UNICA.
    O TODO TIENE EXITO O TODO FRACASA

    -CONSISTENTES: AL FINALIZAR LA TRANSACCION DEBE DEJAR TODOS LOS DATOS EN UN ESTADO COHERENTE, ES DECIR, NO ALTERAR LA ESTRUCTURA DE LOS DATOS 
    PARA MANTENER LA INTEGRIDAD DE LOS MISMOS.
    
    -ISOLADA: LAS CONSULTAS SE GARANTIZAN COMO SI SE HUBIESEN ESCRITO DE MANERA SEQUENCIAL PERO SE EJECUTAN DE MANERA ASINCRONA. UNA TRANSACCION
    NO PUEDE AFECTAR A OTRA TRANSACCION SIMULTANEA, CADA UNA DEBE COMPORTARSE COMO SI FUERA LA UNICA QUE SE ESTA EJECUTANDO EN UN DETERMINADO MOMENTO.

    -DURABLES: UNA VEZ CONCLUIDA LA TRANSACCION, LOS CAMBIOS QUE HAYA PRODUCIDO SON PERMANENTES Y CONFIRMADOS AUN EN EL CASO QUE SE PRODUZCA UN ERROR EN
    EL SISTEMA QUE ALBERGA LA BBDD.
